# V2.2 上下文防护冒烟验证

## 测试材料

- `类型`：超长输入材料，模拟供应商管理系统操作手册
- `预估字数`：约 5000 字
- `内容构成`：
  - 界面操作步骤
  - 业务规则
  - 数据字段
  - 异常处理
  - 混合内容，含纯界面描述与承载业务规则的界面线索

## 验证步骤

### scope 预处理

- [ ] 输入 5000 字操作手册，`scope` 触发预处理
- [ ] 生成 `docs/cache/material-extract.md`
- [ ] 缓存顶部包含元数据：提取来源、是否存在待分类内容、是否允许定向回查原文、提取时间
- [ ] 提取结果不超过 2000 字
- [ ] 纯界面表现被丢弃
- [ ] 承载业务规则的界面线索被保留
- [ ] `scope` 后续分析只基于提取结果，不全文回读原始材料

### sum 预处理继承

- [ ] `sum` 检测到 `docs/cache/material-extract.md` 已存在，直接使用
- [ ] `sum` 读取缓存元数据，正确识别提取来源和待分类内容
- [ ] 项目简报基于提取结果生成，信息完整无丢失
- [ ] 简报中不出现纯界面操作细节

### mind 模块化生成

- [ ] 导图预估规模或复杂度超过阈值时，触发模块化生成
- [ ] 每个模块生成后输出结构化摘要
- [ ] 后续模块生成时只回读摘要，不回读已生成模块全文
- [ ] 全部模块生成后执行跨模块一致性检查
- [ ] 导图前后模块的信息密度无明显差异

### 短材料免处理

- [ ] 输入 2000 字需求描述，`scope` 不触发预处理
- [ ] 输入 3 个模块且无复杂状态分支的简报，`mind` 不触发模块化生成

## 输出要求

验证结果写入本文件，至少补充以下内容：

- 测试材料描述，类型、字数、内容构成
- 每一步输入和实际输出
- 预处理缓存全文，含元数据
- 每个模块的摘要内容
- 前后模块质量对比
- 发现的问题与修复记录
