# 使用说明

## 主入口

- `pm-go`

## 常用命令

- `pm-go`：总入口
- `pm-mm`：思维导图
- `pm-fl`：功能清单
- `pm-ps`：页面清单
- `pm-prd`：PRD
- `pm-rv`：评审
- `pm-fix`：人工改稿后判定
- `pm-pt`：原型
- `pm-pa`：注释原型

## 推荐用法

### 新需求

从 `pm-go` 开始，由系统自动：

- 调用内部技能 `pm-scope`
- 做范围判定
- 判断是否拆子项目
- 推进到后续 PM 阶段

### 已有功能清单，继续做页面清单

调用 `pm-ps`

### 已有功能清单和页面清单，继续写 PRD

调用 `pm-prd`

### 人工改了 PRD

调用 `pm-fix`

### 想检查产物冲突

调用 `pm-rv`

### PRD 稳定后生成原型

调用 `pm-pt`

### PRD 和 Prototype 稳定后生成注释原型

调用 `pm-pa`

## 最短测试路径

真实项目建议按下面顺序试：

1. `pm-go`
2. `pm-fl`
3. `pm-ps`
4. `pm-prd`
5. `pm-rv`

详细测试步骤见：
- `docs/real-project-test.md`

## 手改 PRD 后的推荐做法

### 场景

你已经直接修改了 PRD，不想让系统继续引用旧版锚点，也不想让 AI 擅自重写上游。

### 做法

调用 `pm-fix`，并显式给出当前 PRD 路径。

### 目标

- 识别当前 PRD 是否已替代稳定锚点
- 判断修改只影响 PRD，还是影响页面清单或功能清单
- 给出回写建议，而不是直接全量重写
