# Oh My PM 工作区说明

本目录用于承接从 `ShitPM` 演化出的新辅助器方案，目标是让后续会话直接接手，不再重复追问已定结论。

## 后续会话进入本目录后的默认动作

1. 先读 `README.md`
2. 再读 `DECISIONS.md`
3. 如需理解产品主架构，读 `ARCHITECTURE.md`
4. 如需理解门禁，读 `GATES.md`
5. 如需理解正式交付，读 `DELIVERY.md`
6. 如需理解外部知识协同，读 `KNOWLEDGE.md`

## 已确定的高优先级结论

- 产品名：`Oh My PM`
- 副标题：`牛马拉的赛博磨盘`
- 一句话定位：`一个帮助 PM 在复杂存量系统环境中回应需求、反复对齐并生成正式交付的辅助器。`
- 这不是 `ShitPM 2.0` 的简单改版，而是一个新的辅助器方向
- 主流程不是线性一次过，而是“回应/校验循环 -> 交付前检查 -> 正式交付”
- 交付包仍然保留 `交付型原型 + PRD`
- 原型是评审主展示物和研发第一阅读入口
- PRD 是正式归档主文件，同时补足原型表达不了的规则
- 知识主仓在外部，例如 `Obsidian`，AI 按需读取并回写 Markdown 记忆卡

## 不要重复发散的问题

除非用户明确要求推翻，否则不要重新讨论以下问题：

- 要不要 PRD
- 要不要原型
- 是否应该强行让需求方一次性说清楚
- 是否应该把全部系统预先喂给 AI
- 是否还沿用原 ShitPM 的“多产出物驱动”路线

这些问题已经有结论，后续工作应在既定结论上推进实现。
