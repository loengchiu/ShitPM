# 门禁机制

## 1. 门禁设计原则

门禁锁的不是需求方，而是推进动作。

也就是说：

- 需求方可以继续提需求
- 但不是所有时刻都允许：
  - 对外承诺
  - 开始正式交付
  - 把新增内容直接并入当前版本

一句话概括：

**不阻止需求变化，只阻止在条件不够时进入更重、更贵、更难回头的动作。**

## 2. 四层门禁

### 门禁一：回应门禁

作用：判断当前是否已经能形成一版可信回应。

检查项：

- 当前需求任务明确
- 至少有一版当前版本方案
- 能说清想解决什么问题
- 能说清准备怎么做
- 能给出粗粒度工时或量级判断
- 未确认事实和未澄清问题已显式记录

不过的处理：

- 进入 `内部修正` 或 `补资料`
- 不对外给出像承诺一样的正式回应

#### 回退动作

若是方案表达不清、模块没拆清、工时没法讲：

- 先走 `内部修正`
- 修正后再决定是否需要对齐型原型辅助

若是关键输入缺失：

- 先走 `补资料`
- 资料补齐后再重新判断回应门禁

### 门禁二：对齐推进门禁

作用：判断当前是否值得进入下一轮对齐，而不是低质量散聊。

检查项：

- 本轮变化点明确
- 新增内容已归并到模块或事实状态
- 当前建议明确
- 已能判断新增内容是否影响主结构

不过的处理：

- 先内部整理，不直接开下一轮会
- 必要时补一版对齐型原型，再进入下一轮正式对齐

#### 回退动作

如果问题是：

- 变化点没整理清
- 新增内容还没归并到模块
- 当前建议说不清

则先执行：

1. 内部整理输入
2. 更新本轮变化点
3. 更新当前模块清单和建议
4. 再决定是否发起下一轮对齐

### 门禁三：正式交付门禁

作用：判断能否从回应/校验循环进入正式交付。

#### 前置条件

- 需求方认可当前版本方案
- 没有新增会改变主结构的补充
- 当前方案方向无明显争议
- 模块、工时、排期影响已有稳定判断

#### 交付前检查

至少检查以下六项：

- 目标闭合
- 主流程闭合
- 模块闭合
- 关键未澄清项不会推翻方案
- 预估可讲清
- 正式表达路径明确

不过的处理：

- 根据问题类型，回退到 `内部修正`、`补资料` 或 `重开对齐`
- 不开始交付型原型
- 不开始正式 PRD

#### 回退分流规则

如果没过的原因是：

- 目标不闭合
- 主流程不闭合
- 模块不闭合
- 表达路径不明确

优先回退到 `内部修正`

如果没过的原因是：

- 关键资料缺失
- 关键系统信息缺失
- 关键人反馈缺失

优先回退到 `补资料`

如果没过的原因是：

- 需求方并未真正认可当前版本方案
- 新增内容改变主结构
- 当前方案方向被推翻

优先回退到 `重开对齐`

说明：

- `重开对齐` 不是轮次结果枚举中的一个独立结果值
- 它是正式交付门禁失败后的回退动作
- 一旦执行 `重开对齐`，才会开启下一轮正式对齐，并生成新的轮次编号

### 门禁四：变更门禁

作用：控制正式交付后新增内容的进入方式。

检查项：

- 新增内容属于哪一类：
  - 轻微补充
  - 模块内补充
  - 新模块
  - 主结构变化
- 是否影响：
  - 模块清单
  - 方案预估工时
  - 排期影响
  - 主流程
  - PRD 范围定义

处理规则：

- 轻微补充可并入当前交付，但要记录变化
- 新模块或主结构变化不得默认吞入，必须重开对齐或转为变更

#### 分类标准

##### 轻微补充

满足大多数以下特征：

- 不新增页面或仅改极小文案/字段名
- 不新增流程节点
- 不新增角色
- 不改变模块边界
- 不显著影响工时和排期

##### 模块内补充

满足以下特征：

- 仍在现有模块内
- 不改变主流程主结构
- 可能新增局部字段、局部规则、局部页面元素
- 会带来一定工时变化，但不改变整体模块拆分

##### 新模块

满足以下任一特征：

- 需要新增独立页面组或独立功能块
- 可被单独命名为一个模块
- 会显著改变模块清单和模块级工时
- 可能不改变主流程，但会改变交付范围

##### 主结构变化

满足以下任一特征：

- 改变主流程主链路
- 改变入口或关键角色路径
- 改变系统承接边界
- 改变原定的存量改造 / 新建 / 混合型判断
- 会推翻当前版本方案的大部分结构

#### 判定机制

- AI 先按上述规则给出初判
- PM 结合实际项目情况做最终确认
- 若判断结果影响正式交付范围，以 PM 为准

## 3. 门禁与内部状态的对应关系

### 回应门禁主要读取

- 需求任务
- 当前版本方案
- 未确认事实
- 未澄清问题
- 未澄清原因
- 当前方案预估工时

### 对齐推进门禁主要读取

- 当前版本号
- 本轮变化点
- 当前建议
- 当前模块清单
- 新增资料记录

### 正式交付门禁主要读取

- 已确认事实
- 当前版本方案
- 当前模块清单
- 当前方案预估工时
- 排期影响判断
- 未澄清问题
- 未澄清原因

### 变更门禁主要读取

- 本轮变化点
- 当前模块清单
- 当前方案预估工时
- 排期影响判断

## 4. 一句话总结

需求可以继续提，但回应、承诺、正式交付、变更并入，都要过门禁。
