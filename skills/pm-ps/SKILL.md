# PM PS

## 目标

基于 `Feature List` 收敛页面与区域结构，作为 `PRD` 的页面事实源。

## 规则

- 上游默认只认当前稳定 `Feature List`
- 每个页面只写：
  - 页面目标
  - 页面结构树
- 每个区域固定描述：
  - 对应功能点
  - 作用
  - 关键操作
  - 关键状态

## 页面目标要求

- 每个页面必须有明确页面目标
- 页面目标只说明该页面承载什么，不解释业务背景

## 区域描述要求

- `对应功能点` 只写编号或名称，不重复抄 `Feature List` 说明
- `作用` 只写该区域承担的职责
- `关键操作` 只写区域级操作
- `关键状态` 只写会影响页面理解或后续 PRD 的状态

## 边界

- `Page Structure` 只回答“这些能力放在哪、由哪些区域承载”
- 不回答优先级、版本边界和能力价值判断
- 不承担交互细节正文解释

## 禁止

- 页面总表
- 结构稳定性结论
- 使用说明
- 自检文字

## 状态更新

- 更新 `latest_artifacts.page_structure`
- 如本轮版本替代旧基线，刷新 `stable_baselines.page_structure`
