# PM FL

## 目标

生成稳定功能清单，作为后续 `Page Structure` 和 `PRD` 的能力事实源。

## 输入

- 项目目录
- 思维导图或上游需求结构
- 当前项目简介
- 当前稳定锚点（如存在）

## 规则

- 主体统一采用页面/能力级颗粒度
- 不混用步骤级写法
- 每个模块下补 `涉及角色`
- 主表字段统一为：
  - `#`
  - `功能点`
  - `说明`
  - `优先级`
  - `版本`
  - `PRD章节`
- 优先级使用中文：`高 / 中 / 低`

## 颗粒度要求

- `Feature List` 只写“有什么能力”
- 不写“每一步怎么发生”
- 能力点应能稳定映射到页面或页面能力
- 步骤级内容下沉到 `PRD`、流程说明或原型

## 推荐结构

- 模块结构
- 模块标题
- 涉及角色
- 功能清单主表
- 版本边界
- 关键需求摘要（可选）

## 说明字段要求

- 只写能力边界和范围说明
- 不写页面区域
- 不写按钮级交互
- 不写操作步骤

## 版本边界要求

- 本期范围与后续规划分开写
- 不把“优先级”和“版本边界”混成同一字段

## 关键需求摘要

- 只保留高风险、来源复杂、待确认较多的需求
- 不把完整卡片集重新塞回主文档

## 不负责

- 页面区域说明
- 原型页面映射
- 操作步骤细节

## 状态更新

- 更新 `latest_artifacts.feature_list`
- 如本轮版本替代旧基线，刷新 `stable_baselines.feature_list`
