# PM 思维导图

## 作用

使用这个技能把分析结果转成思维导图，并在二次澄清或访谈后持续回写修订。思维导图不是装饰性产物，而是为了暴露业务结构、范围、主流程、模块、系统结构、待澄清缺口和关键修订点，供后续功能清单继续收敛。

使用 `../../templates/mindmap-spec.md` 作为结构参考。

## 输入

按以下优先级读取：

1. 稳定项目简报
2. 二次访谈补充与修正
3. 来源材料补充说明
4. 当前思维导图版本
5. 当前稳定锚点与最新产物

## 输出

将思维导图产物写入工作项目的 `docs/mindmaps/` 目录。

## 规则

- 思维导图只回答一个问题：这个业务闭环到底是什么样。
- 先产出导图初版；若后续发生二次访谈或澄清，回写到同一导图链路，最终交付“已收敛导图”，而不是额外制造新的长期主产物。
- 导图必须承接项目简报中的背景、范围、角色、关键规则和风险信息，不能脱离分析产物重新发明业务事实。
- 导图必须覆盖：范围、主流程、模块、系统结构、关键判断点、状态分支、风险点、待澄清问题。
- 导图中的“模块”和“系统结构”用于描述业务结构和系统组成关系，不用于描述页面布局或界面区域。
- 不在思维导图阶段承载页面结构、页面区域、按钮位置、弹窗方式、视觉布局等界面实现信息。
- 对仍需继续澄清的节点显式标记 `待确认`。
- 对高优先级节点显式标记 `高优`。
- 对二次访谈后被修正的关键节点显式保留修订痕迹，至少说明“修了什么”以及“为什么修”。
- 如思维导图暴露出问题定义错误、范围结构错误或上游分析不足，允许回退到 `sum`，而不是硬推进到功能清单。
- 思维导图必须足以直接支撑功能清单整理；如果还不能支撑，应继续澄清或回退。

## 状态更新

- 进入本阶段时，将当前阶段设为 `mind`
- 更新 `last_action`
- 更新 `next_recommended` 为“继续澄清”或 `feat`
- 更新 `context_summary`
- 将本轮思维导图写入 `latest_artifacts.mindmap`
- 将仍影响推进的待确认节点同步到 `pending_confirmations`

## 完成定义

- 已形成结构化思维导图，而不是散点列表
- 导图覆盖范围、主流程、模块、系统结构、关键判断点、状态分支、风险点和待澄清问题
- 导图能让读者看清业务闭环，而不是看清页面结构
- 二次访谈或澄清后的关键修订已回写到导图
- 待确认节点已标记清楚
- 若待确认节点足以阻塞功能清单，未将本阶段标记为完成

## 错误处理与回退

- 若项目简报或上游分析产物缺失，回退到 `sum`
- 若思维导图无法从上游产物中提炼出稳定结构，说明原因并停止下游交接
- 若二次访谈结果导致关键分支或范围理解发生明显变化，先回写导图，不直接进入 `feat`
- 若用户要求跳过未澄清的核心分支，说明风险并将状态标记为待确认

## 交接

- 如仍有影响推进的待确认节点，先继续澄清并回写导图
- 如导图已经足够清晰，进入 `feat`
