# Field Clue Rules

用于多阶段共享的字段线索收敛规则。

## 总则

- 字段线索先保留事实，不抢先做正式字段定义
- 允许重名、近义名、不完整描述并存，统一口径由后续阶段收敛
- 只保留真正影响业务理解的字段，不保留纯视觉修饰或技术实现字段

## 优先保留的字段

- 主业务对象字段
- 决定流程分支的字段
- 决定状态变化的字段
- 决定权限可见性或可操作性的字段
- 高频筛选、展示、录入字段

## 阶段职责

- `sum`：沉淀业务对象与关键判断字段线索，不做正式定义
- `mind`：把字段挂到判断节点、状态分支或模块摘要中
- `feat`：把字段保留为能力字段线索，不写正式编号
- `page`：仅在区域写作提示中标明关联字段
- `prd`：统一口径并收敛为正式字段；未采用的线索也要说明去向

## 收敛要求

- `prd` 阶段必须明确每条线索的最终去向：进入数据字典、页面章节、横切规则，或标记为已合并 / 无效线索 / 待确认
- 正文字段定义只写业务可读口径；链路来源统一放入 `AI锚点区`
