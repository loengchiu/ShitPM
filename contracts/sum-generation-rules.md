# Sum Generation Rules

用于 `sum` 阶段的项目简报生成规则。

## 输入与模板

- 事实源优先级：项目目录与上下文、范围澄清结论、来源材料索引、会议纪要、原始需求与补充说明、二次访谈、当前稳定锚点
- 模板优先读取 `shitpm/templates/project-brief-lite.md`
- 相关通用契约优先读取 `shitpm/contracts/artifact-schema.md`、`done-criteria.md`、`error-handling.md`
- 宿主 bundle 解析遵循 `contracts/host-contract.md`

## 模板使用规则

- `project-brief-lite.md` 只是结构骨架，不是可直接交付的半成品
- 模板中的占位项、示例行、空表头和教学说明，只能作为生成脚手架，不能原样出现在最终项目简报
- 生成时必须把模板项替换为真实业务内容；若本期无内容，删除空项或空小节，不得用“暂无”“-”机械占位

## 成稿清洗规则

- 输出最终项目简报前，必须删除所有未替换占位文本、空表格行、空列表项和教学说明
- 禁止在最终正文中保留：
  - `项目 / 需求主题`
  - `背景说明：`
  - `当前问题：`
  - `建设目标：`
  - 只有表头没有真实内容的空表格
  - 只有破折号或空白占位的列表项
- 若某章节没有真实内容，优先删除空项；若整章都无真实内容，删除整章

## 材料预处理

长材料预处理统一遵循 `contracts/material-extraction-rules.md`。

## 目标与边界

- 本阶段交付项目简报，作为进入思维导图前的稳定事实层
- 项目简报至少覆盖：背景、资料分析结论、范围、角色、关键流程文字版、关键规则、风险与待确认项、变更与修正留痕
- 聚焦“项目总览 + 分析结论 + 关键流程文字版 + 留痕”，不要扩写成第二份 PRD
- 信息不足时显式标记待确认，不擅自补全

## 事实分层与字段线索

- 字段线索公共约束遵循 `contracts/field-clue-rules.md`
- `资料分析结论` 必须区分已确认事实、高可信推断、冲突或待核实信息
- 对反复出现的关键业务对象与字段沉淀“字段线索”，供 `mind / feat / prd` 继续收敛
- 本阶段只沉淀字段线索，不做正式定义、不写 `DD-__` 编号
- 只记录主业务对象字段、决定流程/状态/权限的关键字段、高频筛选展示录入字段

## 规则与回退

- 存在二次访谈或补充澄清时，写入补充与修正；不存在时不要伪造
- 若核心问题定义冲突或范围边界互相打架，记录风险并停留在本阶段
- 只有当项目简报足以支撑 `mind`，才允许交接下游
