# Fix Output Checklist

## 输出要求

必须明确输出：

- 本轮实际采用的当前变更产物路径
- 当前变更产物与对应稳定基线的关系
- 修改属于哪一层
- 是否需要回写 `mind / page / feat`
- 是否需要刷新对应 `stable_baselines`
- 推荐下一动作

如本轮已自动改动其他生成物，还必须输出修正记录，至少包含：

- 本轮实际采用的当前变更产物路径
- 当前变更产物与对应稳定基线的关系
- 影响层级
- 影响分析摘要
- 回写粒度
- 冲突来源
- 本轮自动改动了哪些生成物
- 每个生成物的改动原因、改动范围、改动后路径
- 是否刷新了对应 `stable_baselines`
- 哪些事项仍需人工确认

## 禁止事项

- 不得在未判清版本关系前直接引用旧稿
- 不得在未判清影响层级前直接重写上游
- 不得自动改动其他生成物却不留修正记录
- 不得把影响业务流程、判断节点或结构的改动误判成纯文案问题
- 修正记录中不得保留模板占位字段、空表格行、示例值或教学说明
