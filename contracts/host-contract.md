# Host Contract

## 首发宿主

- GitHub Copilot
- Codex

## 宿主最小要求

1. 能暴露 `skills`
2. 能暴露共享 `templates` 与 `contracts`
3. 对当前正式支持宿主，必须暴露 `shitpm-commands`
4. 同时必须能直接调用短技能

## 状态集成能力

若宿主希望让 `pm-go` 输出“真实状态判断”而不是“上下文推断”，至少还需要提供以下能力中的一种可验证来源：

1. 本轮真实读取 `docs/project-status.json`
2. 本轮真实执行 gate 校验
3. 本轮真实更新状态文件

可接受的证据包括：

- 宿主内置机制已明确执行并返回结果
- 当前对话轮中已观察到相关脚本或等效内部机制的真实执行结果
- 用户显式提供了最新状态文件内容，并说明其为当前事实源

若以上证据都不存在，`pm-go` 只能输出“基于上下文 / 基于当前已知状态”的判断，不得表述为已经完成真实状态恢复、真实 gate 或真实状态落盘。

## 最小桥接校验

- `pm-go` 可见
- `pm-prd` 可见
- `pm-rv` 可见
- `shitpm-commands` 可读
- `templates` 可读
- `contracts` 可读

## 可信度口径

- 具备状态集成证据：允许输出“真实状态判断”
- 不具备状态集成证据：默认输出“上下文推断”
