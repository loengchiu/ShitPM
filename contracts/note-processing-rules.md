# Note Processing Rules

用于 `note` 阶段的原型标注规则。

## 目标与输入

- 将稳定版 PRD 的需求点模块化挂载到稳定版 Prototype HTML
- 输出目标是带交互标注层的原型 HTML，不是普通注释文档
- 输入包括稳定版 PRD、稳定版 Prototype HTML，以及用户明确指令：初始化标注或增量更新

## 处理规则

- 先判定当前任务是初始化标注还是增量更新
- 未判定前，不得擅自修改原型 HTML
- 同一组件或模块只保留一个角标
- 默认点击打开，不使用 hover 自动弹出
- 浮窗支持拖拽，只能通过 `X` 关闭
- 页面切换、弹窗打开关闭后必须同步标注状态
- 浮窗内容必须足以替代对应 PRD 阅读
- 纯标注内容问题留在本阶段修订
- 发现需求冲突时回退到 `prd` 或 `mock`

## 模板使用与成稿清洗

- `prototype-annotation.md` 只是标注单元骨架，不是可直接交付的标注文档
- 模板中的占位字段、示例标签和空列表项，只能作为填写脚手架，不能原样进入最终产物
- 输出前必须删除未替换占位文本、空章节、空字段和值为“是 / 否”但未实际判定的示例项
