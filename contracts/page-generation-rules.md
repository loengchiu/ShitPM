# Page Generation Rules

本文件只定义 `page` 阶段的生成规则。

## 输入与定位

- 以上游稳定 Feature List 为直接事实源
- 可用思维导图中的业务闭环信息校验页面划分是否偏离主线
- `Page Structure` 是 PRD 与 Prototype 的共同页面事实源
- 若页面结构设计与业务闭环明显冲突，先标记 `待确认`，不要私自覆盖

## 页面结构规则

- 每个页面只写：
  - 页面目标
  - 页面结构树
- 每个区域固定描述：
  - 对应功能点
  - 作用
  - 包含元素清单
  - 关键操作
  - 关键状态

## 写法要求

- 页面目标必须是一句话，且是用户视角的使用目的，不是功能描述
- `对应功能点` 只写编号或名称，不重复抄功能清单说明
- `作用` 只写该区域承担的职责
- `包含元素清单` 只做盘点，只列有什么，不写行为说明
- 元素类型限定为：按钮、输入框、下拉框、表格、标签、标签页、开关、复选框、单选框、文件上传、日期选择器、搜索框、链接、文本展示、图标、分隔线、弹窗
- 不允许出现“其他”“等”模糊表述
- `关键操作` 只写区域级主操作，控制在 1-3 个
- `关键状态` 只写会影响页面理解或后续需求文档的状态，控制在 1-3 个

## 页面间关系

- 必须显式标注所有页面间的跳转关系（从哪个元素跳到哪个页面，传什么参数）
- 必须显式标注跨页面的数据依赖
- 必须标注多页面共享数据的主数据源
- 页面间关系写入独立的“页面间关系”章节，不散落在各页面描述中

## 写作提示

- 字段线索公共约束遵循 `contracts/field-clue-rules.md`
- 每个区域包含写作提示：关联字段、关键交互、关键校验、关键异常
- `关联字段` 是下游 `prd` 收敛数据字典的直接线索；每个涉及表单、表格、详情展示或筛选条件的区域都必须填写
- 写作提示只列关键项，不列常规项
- 页面级注意事项单独列出（跨区域交互、页面级异常、页面级权限）

## PRD 映射

- 每个页面标注 PRD 对应章节编号（如 5.1、5.2）
- 页面顺序即为 PRD 第 5 章的章节顺序
- 页面总览中列出所有页面的章节映射关系

## 边界与上下游衔接

- 页面结构只回答“这些能力放在哪、由哪些区域承载”
- 不回答优先级、版本边界和能力价值判断
- 不承担交互细节正文解释
- 不脱离思维导图和功能清单自行新增大段页面体系
- 若页面划分无法从功能清单和业务闭环稳定推出，应停在 `待确认`
- 每个区域必须关联至少一个功能点（来自 Feature List）
- Feature List 中的每条功能点必须能在 Page Structure 中找到归属区域
- `prd` 与 `mock` 都应以 `Page Structure` 作为共同锚点
