# Scope Processing Rules

用于 `scope` 阶段的范围探索与拆分判定规则。

## 目标与硬门禁

- 在主链开始前完成最小必要的范围探索和拆分判定
- 在 `scope` 完成并得到用户确认前，不进入 `mind / feat / page / prd`
- 不产出下游正式产物，不把未确认的拆分结论当既定范围

## 材料预处理

材料预处理统一遵循 `contracts/material-extraction-rules.md`。

## 执行方式

1. 先看现有上下文
2. 判断任务是否过大、是否应拆分
3. 信息不足时一次只问一个问题
4. 形成范围结论或拆分方案
5. 用户确认后再交回统一路由入口

## 负责内容

- 澄清当前任务目标
- 判断是否属于单一 PM 任务
- 判断是否需要拆成多个子项目
- 形成轻量项目简介所需的基础信息

## 拆分与交互原则

- 优先问最影响范围边界或拆分判断的问题
- 若需求覆盖多个相对独立的业务域、流程闭环或交付模块，优先拆分
- 拆分时至少给出：拆分方案、每个子项目的目标与边界、依赖关系与推荐顺序、当前优先推进的子项目
- 澄清完成后只保留结论，不保留问答过程

## 问题框架

需要澄清时，按阻塞度选择当前最该问的 1 个问题：

1. 业务目标
2. 用户角色
3. 使用场景
4. 约束条件
5. 成功标准
