# Page Review Checklist

## 完成自审

页面结构生成完毕后，必须执行以下自审；不通过不得进入状态更新：

1. 双向覆盖：Feature List 中每个功能点是否都能在 Page Structure 中找到承载页面和区域？Page Structure 中是否存在没有对应功能点的页面或区域？
2. 页面目标：每个页面是否都有明确的一句话目标，且是用户视角而不是功能视角？
3. 元素完整性：每个区域的元素清单是否完整？是否存在“其他”“等”模糊表述？
4. 页面关系：页面跳转关系、跨页面数据流、主数据源是否已显式标注？
5. 区域归属：每个区域是否都关联了至少一个功能点？是否存在“不知道放哪”的区域？
6. PRD 映射：每个页面是否标注了 PRD 章节编号？页面顺序是否可直接支撑 PRD 第 5 章？
7. 写作提示检查：每个区域的写作提示是否包含具体内容？是否存在元指令？
8. 字段线索检查：所有涉及表单、表格、详情展示或筛选条件的区域，是否都写了 `关联字段`？

## 执行规则

- 自审发现的问题先直接修正
- 若问题仅是页面表达不清，留在当前稿件内修正
- 若问题说明 Feature List 本身存在缺口、重复或能力边界错误，停止推进并回退到 `feat`
