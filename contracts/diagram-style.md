# Diagram Style

ShitPM 的 PRD 图表统一采用接近 Anthropic 技术博客的编辑式图表风格，并输出为 `.drawio` 源文件与 `.svg` 预览文件。

## 输出规则

- 图表源文件：`docs/diagrams/prd/*.drawio`
- 图表预览文件：`docs/diagrams/prd/*.svg`
- PRD 正文只引用 `svg`，不内嵌 draw.io XML
- 图不能替代文字说明；中文流程说明是主规格，图表是结构化辅助

## 生成顺序

1. 先写中文流程/状态/关系说明
2. 从中文说明提取 `DiagramSpec`
3. 选定图表模式
4. 生成 `.drawio`
5. 导出 `.svg`
6. 在 PRD 中引用 `.svg`

## DiagramSpec 最小结构

每张图生成前，必须先形成结构规划：

- `main_claim`：这张图要表达的核心论点
- `pattern`：图表主模式
- `nodes`：节点列表（含中文标签、语义角色）
- `connections`：连线列表（含方向、条件、标签）

## 图表模式

根据内容选择单一主模式，必要时允许少量组合：

- `linear_workflow`：顺序步骤
- `feedback_loop`：重试 / 评估 / 迭代闭环
- `branch_workflow`：判断 / 分支 / 审批关卡
- `grouped_architecture`：系统边界 / 组件 / 包含关系
- `layered_stack`：分层 / 依赖 / 抽象层
- `swimlane_sequence`：多角色交互
- `split_comparison`：前后对比 / 两种方案对照

## 文字规则

- 图中标签统一使用中文，与 PRD 正文保持一致
- 节点标签尽量短，不重复大段说明
- 连线标签只写关键条件或留空，不写长句
- 图中页面、角色、状态、字段名称必须与 PRD 正文一致

## 语义配色

颜色按语义角色分配，不按审美装饰分配：

| 角色 | 填充 | 描边 |
| --- | --- | --- |
| 主流程 / 中性 | `#E6E2DA` | `#8C867F` |
| 上下文 / 资源 | `#EAF4FB` | `#6FA8D6` |
| 控制 / 编排 | `#EEEAF9` | `#9A90D6` |
| 开始 / 触发 | `#F8E9E1` | `#D88966` |
| 结束 / 成功 | `#CFE8D7` | `#71AE88` |
| 警告 / 重置 | `#F3E4DA` | `#C88E6A` |
| 判断 / 分支 | `#E6D7B4` | `#BFA777` |
| AI / 模型 | `#D7E6DC` | `#7FB08F` |
| 未激活 / 禁用 | `#EFECE6` | `#B4AEA6` |
| 错误 | `#F8DFDA` | `#D96B63` |

## 画布与排版

- 画布背景：`#F2EFE8`
- 面板背景：`#FAF8F4`
- 默认画布：`1654 x 930`
- 超高图优先增高，不优先增宽
- 每张图最多 20 个可见节点
- 每张图最多 4 种语义强调色
- 外层必须有薄边框与居中标题

## 节点与连线

- 标准节点：圆角矩形
- 判断节点：菱形
- 开始节点：药丸形
- 结束节点：椭圆 / 圆形
- 所有连线使用开口 V 型箭头：`endArrow=open;endSize=14`
- 连线默认正交路由：`edgeStyle=orthogonalEdgeStyle;rounded=1`
- 不使用阴影、渐变、品牌高饱和色、装饰性图标

## 命名规范

- 图表文件名：`p{两位序号}-{slug}.drawio`
- 预览文件名：`p{两位序号}-{slug}.svg`
- `slug` 使用英文或拼音短横线命名
- 同一张图的 `.drawio` 与 `.svg` 必须同名不同后缀

## 回溯门禁

- 图中每个节点、状态、判断、连线，都必须能在 PRD 同段中文说明中找到依据
- 图中找不到文字依据的内容，必须删除或标记 `[待确认]`
- 若图与正文冲突，以正文为准，并回修图表
