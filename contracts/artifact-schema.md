# Artifact Schema

## Project Brief

核心字段：
- 项目背景
- 范围边界
- 用户与角色
- 关键流程
- 关键规则
- 需求变更记录
- 风险与待确认项

## Feature List

核心字段：
- 模块
- 涉及角色
- 功能点编号
- 功能点名称
- 说明
- 优先级
- 版本
- PRD章节
- 版本边界（可附加）
- 关键需求摘要（可附加）

## Page Structure

核心字段：
- 页面编号
- 页面名称
- 页面目标
- 区域名称
- 对应功能点
- 作用
- 关键操作
- 关键状态

## PRD

核心字段：
- 文档概述
- 范围
- 业务流程
- 用户角色与权限
- 数据字典
- 详细需求说明
- 公共模块说明（如存在）
- 系统依赖 / 非功能要求 / 验收标准汇总 / 建议实施顺序（按需）
- 风险与待确认项
- 页面目标
- 页面区域职责概述（可选；用于帮助纯读 PRD 时建立页面空间感）
- 页面内动作说明（按用户动作组织，以规格说明口径覆盖触发条件、操作过程、操作结果和异常处理；可混用自然段、`1. 2. 3.`、`·`）
- AI锚点区（上游锚点 / 功能点编号映射 / 页面结构引用映射 / 字段线索来源 / 字段线索去向）

## Prototype

核心字段：
- 页面名称
- 页面目标
- 关键区域
- 关键状态
- 关键流程
- 视觉基线
- 页面与区域回溯关系

## Prototype Annotation

核心字段：
- 需求编号
- 模块名称
- 页面定位
- 角标挂载点
- 浮窗标题
- 浮窗正文
- 编号回写位置（可选）
