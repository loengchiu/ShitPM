# PRD Review Checklist

本文件只定义 `prd` 阶段生成完成后的自审与门禁检查。生成完成后读取；不要把写作规则混在这里。

## 一致性校验

全部页面生成完毕后，必须执行独立的一致性校验。

校验清单：
- 功能覆盖：Feature List 中每条“本期”功能点是否都有对应描述
- 页面覆盖：Page Structure 中每个页面是否都有对应章节
- 字段覆盖：Page Structure 各区域写作提示中的 `关联字段`，是否都在 PRD 数据字典或对应页面章节中得到体现
- 字段追溯：`sum`、`mind`、`feat`、`page` 中出现的关键字段线索，是否都能在数据字典中找到去向
- 字段一致性：同一字段在不同页面的展示格式和校验规则是否一致
- 权限一致性：权限矩阵与各页面按钮可见性是否匹配
- 状态一致性：状态流转图与各页面状态筛选/展示是否一致
- 交叉引用完整性：所有 FL-__、PS-__、DD-__ 引用是否指向真实条目
- 场景覆盖：每个区域的 9 类场景是否覆盖了真实存在的关键场景
- 模糊表述检查：全文搜索“需支持”“需考虑”“按规范处理”“详见原型”“同常规”“待定”，确保零命中

## 门禁规则

- 一致性校验不是建议动作，而是生成完成后的必做门禁
- 若存在功能覆盖缺口、页面覆盖缺口、字段覆盖缺口、字段追溯缺口、交叉引用错误、模糊表述命中、关键场景遗漏，不得进入状态更新
- 若问题仅是当前 PRD 表达不足，留在当前稿件修正
- 若问题来自 Feature List 或 Page Structure 缺失或冲突，停止推进并回退到 `feat` 或 `page`

## 角色视角自审

PRD 生成完毕后，或每个页面生成完毕后，必须从以下三个视角审读：

1. 开发视角：是否足以直接写代码
2. 测试视角：是否足以直接编写测试用例
3. 设计视角：是否足以直接支撑原型绘制

若问题涉及上游事实源缺失，不得自行脑补，必须标记 `[待确认]` 或回退上游。
