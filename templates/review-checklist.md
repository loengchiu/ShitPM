# Review Checklist

## 评审基础信息

- 本轮评审对象
- 实际采用文件路径
- 当前路径与 `stable_baselines` 的关系
- 评审模式：自检 / 多角色

## 产物检查

### Mindmap

- 范围、主流程、模块、判断点和待澄清问题是否完整
- 是否存在需要回写 `mind` 的结构性问题

### Feature List

- 是否保持能力级颗粒度
- 是否清楚标注角色、版本边界和跨模块能力

### Page Structure

- 是否只写页面与区域事实
- 页面目标、区域作用、关键操作、关键状态是否完整

### PRD

- 是否基于 `Feature List + Page Structure`
- 是否具体可执行，是否存在空泛表述
- 页面说明是否保持合理结构化
- 通用 B 端能力是否仅在存在特殊规则时展开

### Prototype / Annotation

- 是否与稳定 PRD、Page Structure 一致
- 是否存在需要回退到 `prd / mock` 的冲突

## 结论与路由

- 最终结论：通过 / 带条件通过 / 阻塞
- 是否需要回写
- 是否需要回退
- 推荐下一步动作

## 多角色补充

仅在多角色评审模式下补充：

- 当前评审团组合
- 各角色的主要关注点
- 是否存在需要继续问答的关键问题
