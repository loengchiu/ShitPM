# ShitPM

ShitPM 是一个为 AI IDE 开发环境（如 Antigravity, GitHub Copilot 等）设计的全链路、自动化产品经理（PM）工作流增强工具。它通过内置的状态机控制、严格的阶段锚点和专属的指令集（Skills），将软件需求从早期的概念范围探索，稳妥、高效地推进至项目简介、思维导图、功能清单、页面结构、PRD 以及交互原型和评审阶段。彻底解决并规避 AI 在长文本需求生成中容易产生的幻觉与发散问题。

## ✨ 核心特性

- **🤖 状态机与多极锚点自驱动**：底层基于 `docs/project-status.json` 进行全自动状态调度与流转。严格限定“前一环节产出即为后一环节的绝对锚点（Baseline）”，确保 AI 后续产出不偏离事实源。
- **🛡️ 门禁与阻断机制 (Stage Gates)**：内置瀑布流防御机制。当检测到上游关键节点缺失、逻辑冲突或模块发散时，将自动亮红灯触发阻断，强制人工接入确认。
- **🔄 逆向热更同步 (`pm-fix`)**：深度理解你的意图。支持用户自由修改生成的 Markdown 产物（如增删功能），系统会自动追溯并重构上下游依赖点（如 Feature List 和 Page Structure），保持整体业务生态咬合不断层。
- **🎨 交付级可视化产物闭环**：从业务脉络纯粹的脑图 (`pm-mm`) 到详尽的重工业级 PRD (`pm-prd`)，再到带有悬浮批注的交互原型图 (`pm-pt` / `pm-pa`)，一键生成研发团队可消费的最终物料。

## 🚀 支持的 AI 环境 (Hosts)

目前 ShitPM 脚本安装探测已经或计划支持以下工作环境：
- ✅ GitHub Copilot
- ✅ Codex
- ✅ Antigravity (原生级深度推荐)
- ✅ Cursor
- ✅ Windsurf
- ⏳ Trae (后续评估纳入首发主线)

## 🏗️ 目录结构概述

- `skills/`：PM 核心工作流环节的运转逻辑与 Prompt 定义。
- `commands/`：短命令快捷触发入口（如 `pm-go`, `pm-fl` 等）。
- `templates/`：各环节产物生成的专属 Markdown/HTML 标准模板。
- `contracts/`：上下游工作流节点间必须严格遵守的契约卡点。
- `installers/`：针对各不同 IDE 宿主环境的自动化辅助安装脚本。
- `docs/`：包含白皮书、详细使用手册（`MANUAL.md`）及项目状态机制存储。

## 📦 安装与快速开始

在当前工作区目录下，打开支持的终端，执行以下脚本即可完成初始化部署：

```powershell
powershell -File .\installers\install.ps1
```

该脚本将自动探测你当前已安装的协同 AI 工具，一键完成安装和关联。

安装完毕后，想要立刻启动项目，请向你的 AI 助手（IDE 对话框中）输入：
```text
/pm-go 老板让我在首页加个签到日历。
```

## 🛠️ 指令全家桶全览

整个系统被划分为四个职能小队和相应的审计流程，主要入口包括：

### 1. 调度管家队
- 🟢 `pm-go`：唯一主入口。它可以识别当前你在什么进度，并自动评估引导你进入或拆分下一个正确环节。
- 🔄 `pm-fix`：逆向热同步特性。对你手动爆改的需求文档进行意图识别，自动溯源修复相关的结构锚点。

### 2. 探索定义队
- 🔭 `pm-scope`：业务范围猎犬。探索并收敛出轻量级的项目简介 (Brief)，大项目还会要求切割。
- 🧠 `pm-mm`：结构化脑图梳理业务闭环。只谈流程，不看页面。

### 3. 三重基线锚点队
- 📋 `pm-fl`：生成 **功能清单 (Feature List)**。敲定版本的具体能力边界，这不仅是一张表，更是下发环节不可跨越的鸿沟。
- 🏗️ `pm-ps`：规划 **页面结构 (Page Structure)**。把能力安放到对应页面区域去。只认上游功能清单里的能力，杜绝无脑创造。

### 4. 交付可视队
- 📝 `pm-prd`：执行规格书 PRD 生成器。面对超大型模块可自动切牌分块生成，防熔断。
- 🎨 `pm-pt`：生成带组件风格代码的 HTML 代码级高保真静态重装原型。
- 📌 `pm-pa`：悬浮提示原型的辅助工具，在 HTML 原型上渲染非侵入悬浮角标，随时在页面上看 PRD 条款，画文合一。

### 5. 质量校验（QA）
- ⚖️ `pm-rv`：合规审计。拉起多个视角（业务方、前后端、测试）自问自答对当前产物挑刺，预警 Tiger 和 Elephant 级风险。
