# ShitPM

ShitPM 是一个为 AI IDE 开发环境（如GitHub Copilot、cursor 等）设计的全链路、自动化产品经理（PM）工作流增强工具。它通过内置的状态机控制、严格的阶段锚点和专属的指令集（Skills），将软件需求从早期的概念范围探索，稳妥、高效地推进至项目简介、思维导图、功能清单、页面结构、PRD 以及交互原型和评审阶段。彻底解决并规避 AI 在长文本需求生成中容易产生的幻觉与发散问题。

## ✨ 核心特性

- **🤖 纯自然语言优先**：对普通用户来说，主入口只有 `pm-go`（或别名 `pmgo`）。你只需要像和项目管家说话一样描述当前意图，系统会优先结合当前状态和上下文来判断阶段、解释卡点并推荐下一步。
- **🤖 状态机与多极锚点自驱动**：底层基于 `docs/project-status.json` 进行全自动状态调度与流转。严格限定“前一环节产出即为后一环节的绝对锚点（Baseline）”，确保 AI 后续产出不偏离事实源。
- **🛡️ 门禁与阻断机制 (Stage Gates)**：当检测到上游关键节点缺失、逻辑冲突、未清除阻塞项或待确认项时，系统会先判断当前是否适合继续推进，再用自然语言告诉你为什么不能继续。
- **🔄 逆向热更同步 (`pm-fix`)**：深度理解你的意图。支持用户自由修改生成的 Markdown 产物（如增删功能），系统会自动追溯并重构上下游依赖点（如 Feature List 和 Page Structure），保持整体业务生态咬合不断层。
- **🎨 交付级可视化产物闭环**：从业务脉络纯粹的脑图，到详尽的重工业级 PRD，再到带有悬浮批注的交互原型图，全部由 `pm-go` 在背后调度对应能力完成。

## 🚀 支持的 AI 环境 (Hosts)

目前 ShitPM 脚本安装探测已经或计划支持以下工作环境：
- ✅ GitHub Copilot
- ✅ Codex
- ✅ Antigravity
- ✅ Cursor
- ✅ Windsurf

## 🏗️ 目录结构概述

- `skills/`：PM 核心工作流环节的运转逻辑与 Prompt 定义。
- `commands/`：短命令快捷触发入口（如 `pm-go`, `pm-fl` 等）。
- `templates/`：各环节产物生成的专属 Markdown/HTML 标准模板。
- `contracts/`：上下游工作流节点间必须严格遵守的契约卡点。
- `installers/`：针对各不同 IDE 宿主环境的自动化辅助安装脚本。
- `docs/`：包含白皮书、详细使用手册（`MANUAL.md`）及项目状态机制存储。

## 📦 安装与快速开始

在当前工作区目录下，打开支持的终端，执行以下脚本即可完成初始化部署：

```powershell
powershell -File .\installers\install.ps1
```

该脚本将自动探测你当前已安装的协同 AI 工具，一键完成安装和关联。

安装完毕后，想要立刻启动项目，请向你的 AI 助手（IDE 对话框中）输入：
```text
/pm-go 老板让我在首页加个签到日历。
```

从这里开始，你后续也仍然只需要继续对 `pm-go` 说人话，例如：

```text
/pm-go 现在做到哪个阶段了？
/pm-go 当前能不能继续推进？
/pm-go 继续下一步。
/pm-go 我刚手改了 PRD，帮我判断要不要回写。
```

## 🛠️ 使用方式

对普通用户来说，推荐口径只有一条：**始终从 `pm-go`（或别名 `pmgo`）进入。**
*(注：如果你的 AI 插件不支持识别带连字符的指令，可直接使用无连字符别名 `pmgo`。同理，常用的变更修复也可直接使用别名 `pmfix`，评审阶段可以使用别名 `pmrv`。)*

`pm-go` 会优先在背后完成这些判断：

1. 恢复当前项目状态
2. 判断当前是否可推进
3. 识别你是在查状态、继续推进、进入某阶段，还是做变更修复
4. 在需要推进时先做门禁判断
5. 条件满足时再路由到对应能力

正常情况下，你可以把它理解成“只管说人话，剩下交给 `pm-go`”。  
但也需要知道一条边界：这些结论是否来自**真实状态判断**，仍然取决于当前宿主是否真的完成了状态恢复、门禁校验和状态维护，并且当前轮有可验证的执行证据；如果没有，系统应明确告诉你当前结论更接近基于上下文或现有状态信息的判断。

### 你真正需要记住的只有这几种问法

*(提示：下文所有的 `pm-go` 均可在此基础上平替为 `pmgo`)*

- `pm-go 现在做到哪个阶段了？`
- `pm-go 当前能不能继续推进？`
- `pm-go 继续下一步。`
- `pm-go 进入评审阶段。`
- `pm-go 我刚手改了 PRD，帮我判断要不要回写。`

### 背后的能力仍然存在，但不要求你手记

ShitPM 仍然保留完整链路能力：范围澄清、脑图、功能清单、页面结构、PRD、评审、原型、原型批注、变更修复。

只是现在这些能力默认由 `pm-go` 在背后调度，你不需要再把它们当成普通用户的主流程命令来记忆。
